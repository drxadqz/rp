# S7 Recovery Materials

This directory collects recovery evidence for the RSCD S7 line and nearby historical runs. It is intended for provenance reconstruction, strict checkpoint loading checks, and future reproducibility work.

## Scope Of This Recovery Pass

The search scope for this pass was intentionally limited to:

- `screen_local_physics_field_s8k_from_best`
- the hflip parent directories already known from the S7 lineage

No broader ancestor search was performed in this pass. In particular, a `resume_from` value found inside `screen_local_physics_field_s8k_from_best/protocol.json` is recorded as an untraced earlier ancestor, not followed.

## Contents

- `checkpoint_lineage/`: recovered run folders, evaluation outputs, TTA outputs, checkpoint notes, and derived checkpoint artifacts.
- `historical_source/pre_20260709_training_snapshot/`: compatible recovered source superset. Despite the legacy directory name, this is not a frozen exact source snapshot from one timestamp. It is a recovered superset of code/config/scripts that appears compatible with the historical training line.
- `historical_manifests/`: compressed historical train/val/test manifests and checksums.
- `data_preparation/`: label tables, dataset metadata, canonical class names, and historical preparation scripts.
- `environment/old_training_machine/`: conda/pip/Torch/CUDA/GPU/Python environment records.
- `launch_history/`: selected launch scripts and a curated command record. Full PowerShell history was not exported to avoid unrelated sensitive commands.

## Current Best Publicly Recoverable Model Artifacts

- Formal S7 checkpoint artifacts remain under `checkpoints/c3_farnet_formal_fullmanifest_source_reliable_router_s7_20260709/`.
- A derived recovery artifact was generated at `checkpoint_lineage/recovered_best_model_state_from_vor_teacher.pth`.

Important: `recovered_best_model_state_from_vor_teacher.pth` is not the original historical `best.pt`. It was generated by extracting the model state from the preserved VOR teacher checkpoint and saving it with explicit provenance.

## Recovery Boundary

These materials are suitable for:

- reconstructing the training lineage;
- auditing class mappings and manifest construction;
- checking whether checkpoints strictly load into compatible code;
- documenting what is original, copied, reconstructed, or missing.

They should not be treated as raw private datasets or as proof that every historical command was recovered exactly.
